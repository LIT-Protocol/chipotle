Lit Actions Viem – Examples

Examples for using the viem bundle and LitActions.Viem.toAccount (from 06_litActionsSDK_viem.js) inside Lit Actions. Run these snippets in the dashboard Action Runner (or your own Lit Action) with the appropriate params (e.g. publicKey, address).

LitActions.Viem.toAccount

Create a viem-style custom account that signs with the Lit PKP via LitActions.sign. No private key is used; you pass the wallet address and PKP publicKey. Signature shares are combined by the Lit JS SDK client.

// Create a custom account (address + publicKey; no private key)
const account = LitActions.Viem.toAccount({
  address: params.address,   // e.g. "0x..."
  publicKey: params.publicKey  // PKP public key (hex)
});

// EIP-191 sign message
const sig = await account.signMessage({ message: "Hello, Lit!" });
// Returns op result; full signature is in the client response (by sigName)

// Sign a transaction digest (serialize + hash + sign)
const tx = { to: "0x...", value: 0n, data: "0x", chainId: 1, nonce: 0, gasLimit: 21000n };
const txResult = await account.signTransaction(tx);

// EIP-712 sign typed data
const typedData = {
  domain: { name: "Example", chainId: 1, version: "1" },
  types: { Message: [{ name: "content", type: "string" }] },
  primaryType: "Message",
  message: { content: "Hello" }
};
const typedSig = await account.signTypedData(typedData);

LitActions.setResponse({ response: JSON.stringify({ success: true }) });
  • address — EVM address of the account (0x...).
  • publicKey — PKP public key (hex) used by the sign op.
  • signMessage / signTransaction / signTypedData use EcdsaK256Sha256 and delegate to LitActions.sign.

Viem: Addresses & hashing

The viem bundle (00_viem.js) is available in the Lit Actions runtime. Use it for getAddress, checksums, keccak256, hashMessage, and other utilities.

// In a Lit Action, viem is on globalThis if 00_viem.js is loaded
const { getAddress, getAddress as checksumAddress, keccak256, hashMessage } = (typeof viem !== 'undefined' ? viem : {});

if (typeof getAddress === 'function') {
  const addr = getAddress(params.rawAddress);
  const hash = keccak256(new TextEncoder().encode('data'));
  const msgHash = hashMessage({ message: 'Hello' });
  LitActions.setResponse({ response: JSON.stringify({ address: addr, hash, msgHash }) });
} else {
  LitActions.setResponse({ response: JSON.stringify({ error: 'viem not available' }) });
}

Viem: Encoding & ABI

Encode/decode ABI parameters, function data, and use parseAbi for contract calls.

// encodeAbiParameters, encodeFunctionData, parseAbi (if viem is available)
const v = typeof viem !== 'undefined' ? viem : null;
if (v && v.encodeAbiParameters && v.parseAbi) {
  const types = v.parseAbiParameters('address, uint256');
  const encoded = v.encodeAbiParameters(types, [params.address, 100n]);
  const selector = v.getFunctionSelector('transfer(address,uint256)');
  LitActions.setResponse({ response: JSON.stringify({ encoded: encoded.slice(0, 20) + '...', selector }) });
}

Viem: Units & formatting

Format and parse ether, gwei, and other units.

const v = typeof viem !== 'undefined' ? viem : null;
if (v && v.formatEther && v.parseEther) {
  const wei = v.parseEther('1.5');
  const str = v.formatEther(wei);
  const gwei = v.formatGwei(1000000000n);
  LitActions.setResponse({ response: JSON.stringify({ wei: wei.toString(), str, gwei }) });
}

Full example: Viem.toAccount with message hash

Combine LitActions.Viem.toAccount and (if available) viem’s hashMessage for EIP-191–style signing. This example uses ethers (already in the runtime) for hashing when viem is not used.

const go = async () => {
  const address = params.address || '0x0000000000000000000000000000000000000001';
  const publicKey = params.publicKey;
  if (!publicKey) {
    LitActions.setResponse({ response: JSON.stringify({ error: 'publicKey required' }) });
    return;
  }
  const account = LitActions.Viem.toAccount({ address, publicKey });
  const msg = params.message || 'Hello from Lit Action';
  const result = await account.signMessage({ message: msg });
  LitActions.setResponse({ response: JSON.stringify({ result, message: msg }) });
};
go();
