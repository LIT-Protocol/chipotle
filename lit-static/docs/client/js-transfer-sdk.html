SDK / Transfer API

JavaScript Transfer SDK

Wrapper for transfer endpoints: balance lookups (API key wallet, PKP, or arbitrary address) and sending funds. Use transfer_sdk.js when the Lit Node Simple API has transfer routes mounted.

Tip Balance and send operations depend on the configured chains (e.g. Ethereum, Solana). See the API’s transfer/chain configuration for supported chain values.

Prerequisites

Note
  • baseUrl — API server URL (default http://localhost:8000).
  • Transfer routes — Server must mount get_api_key_balance, get_pkp_balance, get_address_balance, send.
  • API key — From Core SDK getApiKey() when using wallet balance or send.
  • PKP public key — From Core SDK mintPkp(apiKey) when querying PKP balance or sending from a PKP.

Quick Example

import { createTransferClient } from './transfer_sdk.js';

const client = createTransferClient('http://localhost:8000');

const walletBalance = await client.getApiKeyBalance(apiKey, 'Ethereum');

Client & factory

createTransferClient(baseUrl?)
Factory that returns a LitTransferApiClient. Default base URL: http://localhost:8000.
Export: createTransferClient
new LitTransferApiClient({ baseUrl })
Construct transfer client with optional baseUrl. Trailing slashes are stripped.

Balance endpoints

All balance methods return a GetBalanceResponse: { address, balance, chain, symbol }.

getApiKeyBalance(apiKey, chain)
Gets balance for the wallet identified by the API key on the given chain.
GET /get_api_key_balance/<api_key>/<chain>

Parameters: apiKey (string), chain (string, e.g. "Ethereum", "Solana").

Returns: Promise<GetBalanceResponse>

getPkpBalance(pkpPublicKey, chain)
Gets balance for the PKP (programmable key pair) address on the given chain.
GET /get_pkp_balance/<pkp_public_key>/<chain>

Parameters: pkpPublicKey (string), chain (string).

Returns: Promise<GetBalanceResponse>

getAddressBalance(address, chain)
Gets balance for an arbitrary address on the given chain (e.g. 0x... for EVM).
GET /get_address_balance/<address>/<chain>

Parameters: address (string — wallet or contract address), chain (string).

Returns: Promise<GetBalanceResponse>

Send

send({ apiKey, pkpPublicKey, chain, destinationAddress, amount })
Sends funds to a destination address on a chain. The transaction is PKP-signed (authorized by the PKP associated with pkpPublicKey, using the wallet from apiKey).
POST /send

Options:

PropertyTypeDescription
apiKeystringHex-encoded API key (from Core SDK getApiKey)
pkpPublicKeystringPKP public key
chainstringChain identifier (e.g. "Ethereum", "Solana")
destinationAddressstringDestination address
amountstringAmount as string (e.g. wei for EVM)

Returns: Promise<TransferResponse> — { txn_id, success, chain, origin_symbol, origin_amount, gas, timestamp, destination_address }

Response types (summary)

TypeFields
GetBalanceResponseaddress, balance, chain, symbol
TransferResponsetxn_id, success, chain, origin_symbol, origin_amount, gas, timestamp, destination_address

All methods return Promises and throw on non-2xx HTTP responses.
