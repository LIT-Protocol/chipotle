SDK / Core API

JavaScript Core SDK

Wrapper for the Lit Node Simple API v1 core endpoints. Use core_sdk.js for API key management, PKP minting, signing, lit actions, encryption, and signature combination.

Tip Types match core/v1/models (request.rs, response.rs) and core/v1/endpoints.rs. All methods return Promises and throw on non-2xx HTTP responses.

Prerequisites

Note
  • baseUrl — API server URL (default http://localhost:8000).
  • apiKey — From getApiKey() for wallet-scoped operations (mint PKP, sign, encrypt, lit action).

Quick Example

import { createClient, SIGNING_SCHEME_ECDSA_K256_SHA256 } from './core_sdk.js';

const client = createClient('http://localhost:8000');

const { api_key, wallet_address } = await client.getApiKey();
const { pkp_public_key } = await client.mintPkp(api_key);

const signResp = await client.signWithPkp({
  apiKey: api_key,
  pkpPublicKey: pkp_public_key,
  message: 'hello world',
});

const { signature, r, s, recovery_id } = await client.combineSignatureShares({
  apiKey: api_key,
  shareData: signResp,
});

Client & factory

createClient(baseUrl?)
Factory that returns a LitNodeSimpleApiClient. Default base URL: http://localhost:8000.
Export: createClient
new LitNodeSimpleApiClient({ baseUrl })
Construct client with optional baseUrl. Trailing slashes are stripped.

API key & handshake

getApiKey()
Generates and returns a new API key (hex-encoded wallet secret) and wallet address.
GET /get_api_key

Returns: Promise<GetApiKeyResponse> — { api_key, wallet_address }

handshake()
Performs handshake with validators and returns their responses.
GET /handshake

Returns: Promise<HandshakeResponse> — { responses } (array of strings)

PKP: mint & sign

mintPkp(apiKey)
Mints a new PKP for the wallet identified by the API key.
GET /mint_pkp/<api_key>

Parameters: apiKey (string) — Hex-encoded API key from getApiKey().

Returns: Promise<MintPkpResponse> — { pkp_public_key }

signWithPkp({ apiKey, pkpPublicKey, message, signingScheme? })
Signs a message with the given PKP using the provided API key. Uses EcdsaK256Sha256 by default. Pass the full response to combineSignatureShares as shareData.
POST /sign_with_pkp

Options:

PropertyTypeDescription
apiKeystringHex-encoded API key
pkpPublicKeystringPKP public key
messagestringMessage to sign
signingSchemestringOptional; default EcdsaK256Sha256 (use SIGNING_SCHEME_ECDSA_K256_SHA256)

Returns: Promise<SignWithPkpResponse> — { signing_scheme, signed_digest, public_key, share_type, shares, ... }. Pass the whole object as shareData to combineSignatureShares.

combineSignatureShares({ apiKey, shareData })
Combines signature shares. shareData must be the full SignWithPkpResponse object returned by signWithPkp.
POST /combine_signature_shares

Returns: Promise<CombineSignatureSharesResponse> — { signature, signed_data, verifying_key, r, s, recovery_id }

Signing scheme constant

Export SIGNING_SCHEME_ECDSA_K256_SHA256 = 'EcdsaK256Sha256' (secp256k1 + SHA-256). Use when passing signingScheme explicitly.

Lit action

litAction({ apiKey, code, jsParams? })
Executes a lit action with the given JavaScript code and optional JSON params.
POST /lit_action

Options: apiKey (string), code (string — lit action JS source), jsParams (object, optional).

Returns: Promise<LitActionResponses> — { responses } (array of { signatures, response, logs })

Encrypt & decrypt

encrypt({ apiKey, message })
Encrypts a message (time-lock encryption) for the wallet identified by the API key. Store ciphertext and data_to_encrypt_hash for decrypt.
POST /encrypt

Returns: Promise<EncryptResponse> — { ciphertext, data_to_encrypt_hash }

decrypt({ apiKey, shares, ciphertext, dataToEncryptHash })
Decrypts ciphertext using decryption shares from the network/nodes. dataToEncryptHash is the hex SHA-256 hash of the original plaintext (from encrypt response).
POST /decrypt

Options: apiKey, shares (array of hex strings), ciphertext (base64), dataToEncryptHash (hex).

Returns: Promise<DecryptResponse> — { decrypted_text }
